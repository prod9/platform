· pre-install open state

Your platform installation requires an org-level GitHub App.

This server governs delivery for one GitHub org. Create its App once, by hand, on GitHub — then map the credentials into the server's config. The install endpoints stay open until the App is configured and a first user exists; after that they are session-gated.

Map the credentials into config

GitHub shows the App id, private key (once — capture it), webhook secret, and client secret. Feed them via the server's fx config (env; PEM base64 or file). No credential is hand-copied into a UI.

Install on the org & run migrations

Install the App on the org (or its repos), then apply the pending migrations. Each check re-runs on demand — this page converges to green, it never blocks boot. A dirty migration state surfaces as an error, not a button.

Green tests are a baked-in precondition of every build. A failed test suite fails the build — there is no skip-tests opt-out (test-in-build ADR 2026-07-05). The v0.41.8 row above failed on red tests, not a broken image.

Build state is event-sourced over the lifecycle (queue → claim → run → finish/fail); orphaned running rows are reconciled back to queued by a worker against the engine's actual state, never by a staleness timer. Raised-but-unsettled in the walk — shown as design intent.
