# The v0.9 line is platformv2

Date: 2026-07-16
Status: **accepted**

## The ruling

**`v0.9.0` — the templated-init release — is platformv2's first release, and all v2 work
releases into the `v0.9.x` line.** "v2" is the 0.9 line, not an era that spills across
minor bumps: subsequent v2 cuts are `v0.9.1`, `v0.9.2`, … patch releases. A minor bump
past 0.9 is reserved for whatever comes after v2 ships.

The semver strategy has no prerelease channel (`releases/semver.go` — plain `vX.Y.Z`
bumps), so v2's alpha status is carried by this designation, not by an `-alpha` tag
suffix.

## Corrected on the spot

`v0.10.0` was cut on 2026-07-16 as "the alpha" before this ruling was recorded anywhere —
the designation had lived only in conversation and an empty record let the wrong bump
through. The tag was deleted (local + `gh`) within the hour, unconsumed (no image
published, no reference committed), and re-cut as `v0.9.1` at the same commit. The
`v0.9.0` tag itself never moved — version tags stay immutable; only the un-consumed
mistake was withdrawn.

## Consequences

- `release` bumps for v2 work are **patch** (`-p`) until v2 ships.
- `v0.9.1` (= old `v0.10.0` content) is the current v2 alpha: the whole 82-commit rework
  since `v0.9.0` — delivery verbs, render/publish, scaffolding reshape, flux webhook
  delivery, reserved-IP slot.
