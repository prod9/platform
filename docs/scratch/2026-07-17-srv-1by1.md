Output target: **spec/notes only** — a later unattended subagent implements. No code

## Decided

1. **Bare backend routes** — `/api` prefix dropped everywhere. Final surface:
   `GET /health` · `GET /session` · `DELETE /session` · `GET /users/me` ·
   `GET /auth/github` + `GET /auth/github/callback` · `POST /hooks/github` ·
   `GET /builds`. Webui fetch paths + spec ops table move in the same slice.
2. **`GET /session` vs `GET /users/me`** — `/session` = current session (expiry, user
   id; 401 when none), the webui login gate. `/users/me` = own profile (id, name).
   Today's `/api/me` conflation splits.
3. **Manifest-creation flow dropped** (`setup.go`, `github_app` table, `SaveApp`,
   `ErrNoApp` 503 branches all go). GitHub App is created manually on GitHub's UI once;
   credentials arrive via fx config (env; PEM base64/file). Concern renamed
   **install**, not setup.
4. **Rolling-install model** — server always boots (only DB unreachable/unset stays
   fatal). `GET /install` state check drives the webui: App credentials present/valid?
   App installed on org with right permissions (if not: click-to-install button +
   re-check)? Migrations pending (button to run)? Each remediation is a convergent,
   re-runnable action. Migration dirty-state surfaces as an error, not a button.
   Boot-time auto-migrate and boot-time `RequeueOrphans` both removed.
5. **Install gate: no secret, no config** — while uninstalled (no App configured / no
   users) the install endpoints are open; once installed they're session-gated. The
   state check itself is the gate. (INSTALL_SECRET and SECRET-reuse both rejected —
   internal tool.)
6. **Manual App-creation guide, twice-published** — `docs/guides/` how-to AND a webui
   install-page ("Your platform installation requires an org-level GitHub App, steps
   1-2-3"): exact permissions (`contents: write`, `metadata: read`), webhook URL
   (`<server>/hooks/github`) + events, OAuth callback (`<server>/auth/github/callback`),
   restrict to managed org, credential → config mapping. Spec notes the two copies move
   together (or page canonical, guide links; subagent's pick).
7. **Internal queue actions stay granular** — `Create`/`Claim`/`Finish`/`Fail` are
   unit-of-work per general-coding; ruling 2 (big convergent actions) governs the API
   surface only.

## Open items (walk in progress)

- **Item 2 — flux: DECIDED (drop webhook, add observability).** Webhook path dropped:
  GitHub `registry_package` delivery measured ~6m steady / 10-24m surge vs 10m poll —
  webhook wins nothing. Delete `srv/flux` fragment's webhook endpoint + baseline
  Receiver/HMAC Secret/webhook HTTPRoute/`FLUX_HOSTNAME`, supersede the flux-webhook
  ADR citing the measurement, tighten OCIRepository poll to ~2m, drop
  `registry_package` from required App events.
  Replacement: **flux observability** — `GET /repos/{owner}/{repo}/flux` returns the
  repo's delivery state: OCIRepository (ready?, last pulled revision/digest, last
  error) + Kustomization (applied revision, ready condition, last apply error) + k8s
  events on both. Controller-log streaming deferred; events only. Access via the pod
  **ServiceAccount** per the in-cluster control-plane ADR (2026-06-14) — no kubeconfig,
  ever; baseline grants the SA read-only RBAC on Flux CRDs + events; off-cluster serve
  has no flux view. Repo→flux-object mapping pinned in the spec by the baseline's
  naming convention, not guessed at runtime.
- **Item 3 — DECIDED: platform workers reconcile against the engine.** Background
  layer becomes fx `worker` module jobs. A recurring reconcile job nudges build rows to
  match the dagger engine's actual state — a `running` row with no live engine
  counterpart is by definition an orphan → requeue; truth from the engine, never
  bookkeeping (no staleness timers, no in-memory ids, no leases). Runner
  (claim/run/record) reshapes as worker jobs; boot does nothing but start the worker.
  Subagent details the engine-state query via the existing `engine` package.
- **Migration trigger** — decided: install-screen button (item 4). `platform migrate`
  headless CLI subcommand: shelved unless chakrit wants it.
