## Slice C — designed, NOT started

Goal: the CUE module path becomes a **scaffold-time required input** the operator supplies
at init, flowing to `cue.mod` + app import lines, **never** persisted to `platform.toml`.
Manual `cue.mod` editing is **banned** — the value must be asked, not placeholdered.

Design (all settled with operator; drive from here):

1. **`Framework.RequiredScaffoldInputs(wd) []string`** — NEW interface method. Discovery
   declares the input var-names to prompt. Infra returns `["CUE_MOD_PREFIX"]` **only when
   greenfield** (`!cuemod.Present(wd)`); the 6 others return `nil` (embed a no-op base to
   avoid boilerplate). Plain `[]string` — the var-name IS the prompt label
   (`fx prompts.Str("CUE_MOD_PREFIX")` shows `CUE_MOD_PREFIX:`). NOT a `{Key,Question}`
   struct. Name is deliberately NOT "…Vars" — `[vars]` is the live persisted render table;
   these are scaffold inputs, off `Project.Vars`.
2. **The framework owns resolution.** All three template holes (`ModulePath`,
   `DaggerVersion`, `ImageBase`) live only in `framework/infrabase/` (Infra's files), and
   `Resolve` has one production callsite (`planSpecFiles`), a no-op for the other 6
   frameworks (empty `Files`). So move the `CUE_MOD_PREFIX → ModulePath` routing + the
   greenfield-input-vs-existing-`cue.mod` decision + `Data` assembly **into Infra**.
   `planSpecFiles` goes dumb (no `Data`, no `ModulePath`, no `cuemod` read). Two shapes to
   pick at code time (lean B):
   - (A) add `ScaffoldData(wd, repository, inputs) (scaffold.Data, error)` method; keep the
     generic `Resolve` in `planSpecFiles` calling `fw.ScaffoldData(...)`.
   - (B) `Scaffold` takes `inputs` and returns **resolved** Files; `Resolve` becomes
     Infra-internal; `planSpecFiles` just carries `spec.Files`. Needs `repository` threaded
     to Scaffold (for `ImageBase`).
3. **`runInit` reorders** to discover → `RequiredScaffoldInputs` → prompt each into a
   `map[string]string` → pass framework + inputs into `Analyze` (thread the discovered `fw`
   so `Analyze` doesn't re-discover). Zero framework-specific code in the prompt loop.
   Validation (`CUE_MOD_PREFIX` has a dot in the first segment) lives **framework-side**
   (Infra, where the module path is built) — fail clearly, not CUE's cryptic error.
4. **Delete `import_prefix` end-to-end**: `Project.ImportPrefix` + toml key,
   `GenerateInfo.ImportPrefix` + the `Generate` write, `Spec.ImportPrefix`, Infra's
   `example.com` seed. Grep-confirmed the platform.toml key is already **write-only** (read
   by nobody) — the real module-path source at init was `spec.ImportPrefix` (greenfield) or
   `cuemod.Path` (existing).
5. **`daggerVersion` seam**: if resolution moves into `framework` (Infra), the cmd-side
   `var daggerVersion = framework.DaggerVersion` seam (`init_plan.go`) can go — `framework`
   has its own `readBuildInfo` seam. Check that the init_plan tests re-point to it.
6. **ADR rewrite**: `docs/decisions/2026-07-12-cue-module-path-via-import-prefix.md`.
   Durable insight survives (module path ≠ repository, separate namespaces). Reshape to:
   init-slot `CUE_MOD_PREFIX`, `cue.mod` is sole home, render loads by cue.mod module, no
   persisted key, no manual cue.mod edit. Delete the "Deferred" section. Rename the file /
   concept off `import_prefix`.
7. **In-slice smoke**: `tests.cue` infra-init init line gains the `CUE_MOD_PREFIX` positional
   arg (a valid dotted path, e.g. `example.com`); the infra-init `platform.toml` golden drops
   `import_prefix` → re-record. `init_plan_test.go` `TestAnalyze_infraGetsBaselineUniformly`
   asserts `import_prefix`/`module:` — update for the new flow.

## Watch (design laws that bit this session)

- **No app-vs-infra branch in `init`** (`init.go` comment is a commandment). Infra's
  differences are expressed as framework-declared data (like the now-removed `NeedsGitRepo`,
  like `RequiredScaffoldInputs`), consumed generically. Never an `IsInfra` predicate.
- **Narrow cut**: `RequiredScaffoldInputs` has ONE real consumer today (greenfield Infra);
  Go/pnpm onboard existing module files, they don't scaffold fresh ones. Keep it minimal;
  generalize to a `[]Need`/map when a second consumer is real.
