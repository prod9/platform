## Slice 3 — `GoBasic` test opt-out: WONTFIX (2026-07-05, committed `243e9a9`)

The opt-out is dropped. It rested on a **conflation** of platform's two test suites, and it
contradicts the testing philosophy now recorded as a per-repo Law (CLAUDE.md `### Testing`)
and in the [test-in-build ADR](../decisions/2026-07-05-test-in-build-is-a-hard-gate.md):

- **`go test ./...`** — hermetic unit tests (no docker/network, fresh-clone runnable). Runs
  **inside every image build** (the `Go*` builder gate) and locally. This is the suite in
  `go_basic.go`; verified hermetic — no test imports Dagger, opens a socket, or hits network.
- **`./test.sh`** — blackbox smoke (`chakrit/smoke`), needs docker. Runs **on the host**,
  manually / pre-deploy — the drift detector. Never runs inside the build.

The earlier "in-build test blocks self-build" claim conflated the first with the second.
Platform's `go test ./...` runs fine docker-free, so **self-build was never blocked**.
Test-in-build is a hard, non-configurable gate — blackbox-first, no skip-tests option ever.
Heavier per-project setup, when needed, goes through opt-in **build hooks** (backlog below),
never a gate opt-out.
